Introduction to SQL

SQL is a standard language for accessing and manipulating databases. Using SQL, you can insert records, update records and delete records. You can also create new database objects such as databases and tables.

More advanced features include creating stored procedures (self contained scripts), views (pre-made queries), and setting permissions on database objects (such as tables, stored procedures, and views).

So, what can SQL do?

  • SQL can execute queries against a database
  • SQL can retrieve data from a database
  • SQL can insert records in a database
  • SQL can update records in a database
  • SQL can delete records from a database
  • SQL can create new databases
  • SQL can create new tables in a database
  • SQL can create stored procedures in a database
  • SQL can create views in a database
  • SQL can set permissions on tables, procedures, and views

Common Uses

SQL is used to manage databases. Data Analysts will use SQL to query tables of data and derive insights from it. Data Scientists will use SQL to load data into their models. Data Engineers and database admnistrators will use SQL to ensure that everybody in their company has easy and intuitive access to the data they need.

Most websites will have a SQL backend that automatically stores everything from user details to user engagement across a particular app.

Creating and Droping a Database

To get started, we going to create a new database writing some SQL code, to this we are going to create a database that is going to contain some data about countries, so we called hello_world, so to create the database we write something like this: CREATE DATABASE hello_world; Creating a database does not select it for use; you must do that explicitly. To make hello_world the current database, you have to use this statement: USE hello_world; So, to drop a database, you have to use the next code: DROP DATABASE hello_world;

Creating and Droping a Table

To store some data on our database, we need to create a table, a table is a collection of related data held in a table format within a database, that consists of columns and rows. For this example, we are going to create something that looks like this:

You may have noticed that after the name of the column there are data definitions, this is because every column has and specific type of data that can hold, so the following are examples of the most common field types in SQL:

DATEA date

Then to drop a table, the syntax is as follows: DROP TABLE countries;

SELECT Basics

The most simple way to retrieve data from a table is with the next code: SELECT * FROM countries; This will retrieves everything from a table. This form is useful when you want to review your entire table, for example, after you've loaded it with your initial data set.
